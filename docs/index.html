SpecTracer takes your Gherkin feature files as the source of truth and collates Unit, Integration, and E2E test results into a single, self-contained HTML report — plus an optional schema-validated JSON report for scripts and CI tooling. Tech-stack agnostic.

Why SpecTracer?

Modern teams test across multiple layers — but visibility is fragmented. SpecTracer brings everything together.

☰

Fragmented Visibility

Unit, integration, and E2E tests live in different directories (or repos). There is no single view of overall coverage — until now.

▲

Inverted Pyramids

Teams unknowingly accumulate slow E2E tests instead of fast unit tests. The pyramid dashboard makes it obvious at a glance.

⚖

No Traceability

Hard to know if a specific business scenario is tested across all necessary layers. SpecTracer flags missing coverage per scenario.

⚙

Tooling Lock-in

Most reporting tools are tied to one framework (Allure for Java, Cypress Dashboard for Cypress). SpecTracer works with any stack.

How It Works

Three steps from feature files to a rich HTML report.

1

Write Gherkin Feature Files

Define your scenarios in standard .feature files with tags like @FC-42 that link to test results. Declare required layers with @require-unit, @require-e2e, or module forms like @require-e2e:checkout.

2

Run Your Tests

Use any test framework — pytest, Jest, Cypress, Behave, anything that produces JUnit XML or Cucumber JSON. SpecTracer doesn't care what language you use.

3

Generate the Report

Run uv run spec-tracer (or spec-tracer after pip install). SpecTracer links tests to scenarios via shared tags and produces a single self-contained HTML report with five interactive sections. Add output_json to your config to also emit a schema-validated JSON report for scripts and CI tooling.

See It in Action

A live report generated from SpecTracer's own test suite — showing the Coverage Progress dashboard, Test Pyramid, and Feature Traceability tree with completion bars, outcome pills, and searchable columns.

Quick Start

Get a report generated in under a minute.

Install & run
# Install
pip install spec-tracer

# Write Gherkin feature files in ./features/
# Tag each scenario with an ID like @FC-1

# Run your tests with JUnit XML or Cucumber JSON output
pytest tests/ --junitxml=reports/tests.xml
behave features/ --format json -o reports/e2e.json

# Create spectracer.config.json:
cat > spectracer.config.json <<EOF
{
  "features": ["./features"],
  "unit": { "": ["./reports/tests.xml"] },
  "e2e": { "": ["./reports/e2e.json"] },
  "output": "./reports/spectracer-report.html"
}
EOF

# Generate the report
spec-tracer

# Open reports/spectracer-report.html in your browser
ℹ Config-driven CLI All configuration lives in spectracer.config.json. Run spec-tracer with no arguments to auto-discover it, or pass a path: spec-tracer path/to/config.json. No flags needed.
ℹ Want machine-readable output too? Add "output_json": "./reports/spectracer-report.json" to your config. It's optional, additive, and validated against the shipped JSON Schema — perfect for PR bots, custom CI gates, or dashboards. See JSON Report for details.

What You Get

The generated report is a single HTML file with all CSS and JS inlined (a CDN font loads gracefully with a system fallback) and five sections — plus an optional JSON report carrying the same data for tooling.

●

Coverage Progress

The headline metric: Tested: X / Y (Z%). Per-feature breakdowns with green/amber/red thresholds. Designed for daily standups.

▲

Test Pyramid

3-tier visualization of E2E / Integration / Unit with test counts, duration, and pass rates. Spot inverted pyramids instantly.

☰

Feature Traceability

Searchable tree: Feature → Scenario → Layer results. Full Gherkin text, required layer (and optional module) status, and per-test pass/fail/skip.

⚠

Failure Breakdown

Every failed test across all layers with feature/scenario context and expandable stack traces. One place to triage everything.

⚙

Health Checks

Progress, pyramid ratio, and End to end Runtime checks with configurable thresholds. Visual indicators that make team health obvious.

⚐

Unlinked Tests

Test results whose tags don't match any scenario. Catch orphaned or mis-tagged tests that would otherwise go unnoticed.

{ }

JSON Report

Set output_json and get the same coverage, pyramid, health, and traceability data as schema-validated JSON — built from the same data model as the HTML report, so the two never drift. Post summary to your own dashboard in a couple of CI lines to track trends over time (see CI/CD guide).

📦

Module Scope

Pin requirements to code areas with @require-unit:auth or @require-e2e:checkout. Unit, integration, and E2E all use the same module-keyed config shape.

Trusted by the Team Behind It

SpecTracer dogfoods its own reports — it processes its own test outputs to generate a self-report. Every feature is validated with real Gherkin scenarios and E2E tests before it ships.
